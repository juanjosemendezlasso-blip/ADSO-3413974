PROYECTO: SISTEMA DE TOMA DE ASISTENCIA

INGENIERÍA DE REQUERIMIENTOS

Descripción

El proyecto tiene como objetivo optimizar el tiempo que emplea el instructor en el registro de la asistencia de los aprendices, reemplazando el proceso manual por un sistema que permita realizar el registro de forma rápida, organizada y segura.


Objetivo General

Desarrollar un sistema que permita a los instructores del SENA registrar y consultar la asistencia de los aprendices de manera eficiente, reduciendo el tiempo empleado en este proceso.

Objetivos Específicos
Permitir al instructor iniciar sesión en el sistema.
Consultar las fichas de formación asignadas.
Registrar la asistencia de los aprendices.
Almacenar el historial de asistencia.
Generar reportes para el seguimiento académico.

Ingeniería de Requerimientos
Instrumentos de Recolección de Información

Para identificar las necesidades del sistema se utilizarán los siguientes instrumentos:

Entrevistas a instructores.
Encuestas a aprendices.
Observación directa del proceso de asistencia.
Revisión de los procedimientos institucionales del SENA.

RF01. El sistema debe mostrar la lista de aprendices de la ficha seleccionada para realizar el registro de asistencia.
RF02	El sistema debe mostrar las fichas asignadas al instructor.
RF03	El instructor debe poder seleccionar una ficha de formación.
RF04	El sistema debe mostrar la lista de aprendices matriculados.
RF05	El instructor debe registrar la asistencia de cada aprendiz.
RF06	El sistema debe guardar automáticamente el registro de asistencia.
RF07	El instructor debe consultar el historial de asistencia.
RF08	El coordinador académico debe generar reportes de asistencia.

Caso de Uso

Tomar Asistencia

Actor Principal

Instructor

Actores Secundarios
Coordinador Académico
Aprendiz (consulta de asistencia)
Descripción

El instructor registra la asistencia diaria de los aprendices pertenecientes a una ficha de formación.

Precondiciones
El instructor debe haber iniciado sesión.
Debe existir una ficha asignada.
Flujo Principal
El instructor inicia sesión.
El sistema valida las credenciales.
El instructor selecciona la ficha de formación.
El sistema muestra el listado de aprendices.
El instructor registra la asistencia.
El sistema guarda la información.
El sistema confirma el registro exitoso.
Flujo Alternativo

Si ocurre un error de conexión, el sistema informa al instructor y no guarda los cambios.

Postcondición

La asistencia queda registrada en la base de datos.

Actores
Instructor

Responsable de registrar la asistencia de los aprendices.

Coordinador Académico

Consulta estadísticas y genera reportes.

Aprendiz

Puede consultar su historial de asistencia.

Especificación SRS (IEEE 830)
Introducción

Este documento define los requisitos para el Sistema de Gestión de Asistencia del SENA, cuyo propósito es optimizar el registro diario de asistencia de los aprendices.

Propósito

Desarrollar una aplicación que facilite el registro, consulta y control de asistencia de los aprendices.

Alcance

El sistema permitirá:

Autenticación de usuarios.
Consulta de fichas de formación.
Registro de asistencia.
Consulta del historial.
Generación de reportes.
Administración de usuarios.
Usuarios del Sistema
Instructor.
Coordinador Académico.
Administrador del sistema.
Aprendiz (consulta de asistencia).
Restricciones
Acceso mediante credenciales institucionales.
Disponibilidad durante la jornada de formación.
Cumplimiento de las políticas de seguridad del SENA.
